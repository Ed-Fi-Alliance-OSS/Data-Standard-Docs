Repeat State Assessment Failures

Background

Campus Dashboard Metrics

Whether or not one believes standardized test performance is indicative of how well schools are preparing students for postsecondary education and the workforce, every performance management dashboard must include state accountability test results due to their high-stakes nature for schools as well as individual students. This is particularly true in the case of students who have failed to meet standards for more than one year. Reinforcing this, in focus groups conducted with educators, standardized test performance was consistently named as one of three top data needs. Similarly, the Repeat state assessment Failure metric, as presented in screen shots of the campus dashboard, was considered useful by 96% of those surveyed. In particular, educators valued having historical and detailed standardized test data in one place.

Although examinations vary by state, the general principles applied in the campus dashboard and data model for standardized test performance would be applicable to any state.

Primary Metric

  • Repeat state assessment Failures: percent of students failing state assessment for two consecutive years.

Related Metrics

  • State Assessment Performance: percent of students passing state assessment in all tested subjects: reading/ELA, writing, math, science and social studies at the applicable grade levels and for all tests. Aggregated performance on alternative tests, including those for special education and Limited English Proficient students, is also reported.      
  • State Assessment Exit-Level Performance: percent of students passing the exit-level state assessment required for graduation (first administered in the 11th grade).

Business Rules

The Repeat state assessment Failure metric displays the percentage of students that did not meet the Met Standard criteria for the same state assessment subject for two consecutive administrations. State assessment taken with accommodations are included in the Repeat state assessment Failure metric, for the following:

  • State assessment-S is the standard state assessment test where the administration language is Spanish.
  • LAT is the standard state assessment test with Linguistic accommodations.
  • State assessment-Accommodated is the standard state assessment test with special accommodations.

If an alternate state assessment test version was taken, the metric is displayed for each state assessment test version by subject area in a separate section and is not counted in the standard state assessment percentages, for the following:

  • State assessment-M is a special state assessment test for learning disabled students.
  • State assessment-Alt is a special state assessment test for severely disabled students.

For most of the school year, the values reflect the prior year state assessment scores. Once the current year state assessment test results are received and loaded, the display is updated.

If the student took the same state assessment test multiple times in the same year, the last score is used. If the student failed a state assessment , but retook it in the same school year and passed, the first test is not counted as a failure for the year.

As a student must have two years of standardized test data, the metric should only include 4th grade and above.

Data Assumptions

  • Student state assessment scores by subject are available for the grade level as listed in Table 6.
  • The state-defined thresholds of “Met Standard” and “Commended” are indicated in the state assessment results file.
  • The type of the state assessment test is available per student (e.g., state assessment-M, state assessment-Alt, etc.).
  • Student absence and reason for not taking a particular test are indicated in the state assessment results file.
  • Historical test scores for each student are available in the test provider files for the current year. If these are not available in current-year files, developers must acquire test files from prior years to obtain historical scores.

Computed Values

This campus metric is a roll-up for the cohort of current students, across all grades in the campus, as further elaborated in this section.

The campus metric is based on the last set of state assessment scores loaded, as follows:

  • For the majority of the year, the previous year’s state assessment scores are reflected
  • Toward the end of the end, the current year’s state assessment scores may be loaded

If the student failed one or more of the last set of state assessment, the previous administration of that subject’s state assessment(per Table 4) is checked to see if the student failed it also.

Footnotes

An icon appears next to any metric where students have been excluded.  When the user hovers over the icon, count of students who have been excluded appear in a tooltip with the reason(s) for exclusion.  By clicking on the icon, the data administrator will link to the data administrator’s page for more detail.

Best practice is to exclude and footnote students based on the following criteria:

  • Non-participation
  • Missing test data
  • Alternate testing

As a student must have two years of standardized test data, the footnote should only include 4th grade and above.

Best practice is for the date of last data refresh to appear next the metric in the following format:

  • (Through April 2010) –container level
  • (2010) – granular level

Implementation Considerations

Student Identity

Maintaining a correct and consistent student identity is at the center of any education data system. Most systems use some sort of unique identifier. However, sometimes this identifier is entered incorrectly or sometimes different systems use different identifiers.

The UDM XML supports the interchange of multiple types of identifiers. The StudentReference is a complex type within the UDM to maintain the referential integrity of the student (that is, ensuring that the data associated with each student is accurately associated with the right student). The complex type of the student reference assists with implementing the accurate matching algorithm to identify a student by utilizing any of the individual attributes (e.g., Student Unique State ID, Student ID, Campus Local ID (with Campus ID), Name and Birth Date). For example, if the Student Unique State ID is unknown, you can find the student’s identity by their Student ID, First Name, Last Name and Birth Date.

Reporting Different Versions of State Assessment Tests

Best practice is for a developer to report each of the different test versions for students with special situations as separate metrics. A student may, in any single grade level, take different versions of a state assessment test for different subjects and these should be distinguished so an educator can track progress of students.

Reporting Retakes of the Same State Assessment Test

Currently the metric reports the most recent result for a given state assessment test by type, grade level, and subject. Thus, if a student failed a state assessment test in a subject, but retook it in the same school year and passed, the first time is not counted as a failure for the year.

Loading of Current Year State Assessment for Consistent Display

Ed-Fi currently computes the metric for the cohort based upon the last state assessment taken by each student. It is important that a new year’s state assessment all be loaded at once in order to maintain the consistency of the metric.

Changes in State Assessments

If states are undergoing changes in assessment standards and/or tests, it may impact the way that current and longitudinal data is presented. 

Aggregate Metric Inclusions and Exclusions of Special Education Students

Each campus's aggregate metrics may include or exclude the special education population, depending on the intended purpose for a specific metric and campus goal.

The Historical Graph allows the user to see historical data across multiple timeframes.  As time passes, there are many bars to be displayed; therefore, the slider on the bottom allows the user to adjust the number of bars displayed as well as the time period shown.  The timeframes for this metric were chosen based on educator feedback.
